# Build Script: Generate Distribution Zip Files

## Context

The project needs a way to create distribution zip files for team members. Currently, users must have Python pre-installed and the `run_*.bat` files contain hardcoded absolute paths from Ajay's machine. We need a single build script (`build_dist.bat`) that generates 3 zip types, and an updated `install.bat` that auto-detects Python (embedded or system) and regenerates wrapper scripts with correct paths.

---

## Files to Create

### 1. `build_dist.bat` (~120 lines)

A menu-driven build script with 3 options:

```
[1] Windows + Embedded Python  (~40-50MB zip)
[2] Windows (Python required)  (~200KB zip)
[3] Mac/Linux                  (~200KB zip)
[A] Build all three
```

**Option 1 (Windows + Embedded Python):**
1. Download `python-3.12.8-embed-amd64.zip` from python.org to `build_tmp/`
2. Extract to `build_tmp/dist_win_full/python/`
3. Enable pip: uncomment `import site` in `python312._pth`, download `get-pip.py`, run it
4. Install deps: `python.exe -m pip install -r requirements.txt --target ../lib`
5. Configure `python312._pth` to add `../lib` and `../` paths
6. Remove pip from embedded Python (not needed at runtime, saves ~5MB)
7. Copy all distribution files into `build_tmp/dist_win_full/`
8. Zip as `dist/TempoAutomation-vX.X-Windows-Full.zip`

**Option 2 (Windows, no Python):**
1. Copy distribution files to `build_tmp/dist_win_lite/`
2. Zip as `dist/TempoAutomation-vX.X-Windows-Lite.zip`

**Option 3 (Mac):**
1. Copy distribution files to `build_tmp/dist_mac/`
2. Exclude: `*.bat`, `confirm_and_run.py` (Windows-only: ctypes.windll)
3. Include: `install.sh`
4. Zip as `dist/TempoAutomation-vX.X-Mac.zip`

**Distribution files (common):**
- `tempo_automation.py`, `tray_app.py`, `confirm_and_run.py` (Win only)
- `config_template.json`, `org_holidays.json`
- `requirements.txt`, `README.md`
- `install.bat` (Win) / `install.sh` (Mac)
- `run_daily.bat`, `run_weekly.bat`, `run_monthly.bat` (Win only)
- `assets/favicon.ico`
- `examples/*.json` (3 role configs)
- `docs/guides/*` (4 user docs)

---

### 2. Update `install.bat`

**Add Python auto-detection** at the top (before step 1):

```
Check order:
1. %SCRIPT_DIR%python\python.exe  (embedded, shipped in zip)
2. python.exe in PATH             (system Python)
3. Neither в†’ error with download link
```

Set `PYTHON_EXE` and `PYTHONW_EXE` variables, use them throughout.

**Step 2 (pip install):** Skip if `lib/` directory exists (embedded zip already has deps).

**Step 5 (scheduled tasks):** Regenerate `run_daily.bat`, `run_weekly.bat`, `run_monthly.bat` with the detected Python path and actual `%SCRIPT_DIR%` before creating schtasks. This replaces the hardcoded paths.

## Embedded Python Details

- **Version:** Python 3.12.8 (latest stable 3.12.x, well-supported by all deps)
- **Download URL:** `https://www.python.org/ftp/python/3.12.8/python-3.12.8-embed-amd64.zip`
- **`python312._pth` config after build:**
  ```
  python312.zip
  ../lib
  ../
  .
  import site
  ```
- **Why not 3.14:** PyInstaller, some deps may not support it yet. 3.12 is safe.
- **Why `--target lib/`:** Keeps deps separate from embedded Python, clean structure.

## Verification

1. **Build all 3 zips:** Run `build_dist.bat`, select `A`
2. **Windows Full:** Extract to a folder, run `install.bat` -- should detect embedded Python, skip pip install, complete all 7 steps
3. **Windows Lite:** Extract, run `install.bat` -- should detect system Python, run pip install, complete all 7 steps
4. **Mac:** Extract on Mac, run `chmod +x install.sh && ./install.sh` -- should work with system python3
5. **Verify run_*.bat regeneration:** After install, check that `run_daily.bat` contains correct Python path (not Ajay's hardcoded path)
6. **Verify tray app launch:** After install, tray icon should appear
